## 범위

- `commands/ohd-new-project.md` — 인터뷰를 진행하고 답을 플래그로 모아
  스크립트를 호출하는 커맨드 프로즈. 스크립트 위치는
  `${CLAUDE_PLUGIN_ROOT}/assets/new-project.sh` (기존 관례). 이름은 `ohd-`
  접두어 규약 준수 (`new-project`는 범용 단어 — CLAUDE.md §Conventions).
- `assets/new-project.sh` — 결정론적 스캐폴더 (모든 입력 플래그, 인터랙션 0).
- `assets/hosts-templates/{manifest,setup.sh,README.md}`.
- `assets/CLAUDE.md.template`.
- `tests/new-project-smoke.sh` + CI.
- way-of-working 라우팅 표에 "새 연구 프로젝트 시작 → `/ohd-new-project`" 행.
- 제외 (명시): ohd-share 실행 스킬(백로그), 다중 hosts 동시 스캐폴드(1개 max,
  추가는 수동), GitHub org 고급 설정, **SessionStart 브리프 훅 스캐폴드**
  (우산 spec의 검토 항목 — v0.3 제외, 백로그 유지).

## 템플릿 치환 메커니즘 (전 템플릿 공통 규칙)

- `campaign.sh` **및 `install-hooks.sh`**: config 헤더의 `${CAMPAIGN_X:-기본값}`
  에서 **기본값 부분만** sed로 치환 (install-hooks.sh의 TRUNK 포함 — 안 하면
  다른 클론에서의 재실행이 main만 보호하는 훅을 조용히 설치) — 런타임 env
  오버라이드 계약(v0.2 드롭인 가이드) 보존.
- `CLAUDE.md.template` / `hosts-templates/*`: `{{UPPER_SNAKE}}` 토큰을
  new-project.sh가 치환. 토큰 목록은 각 템플릿 상단 주석에 열거.
- **동적/조건부 내용** (머신×환경 매트릭스의 가변 행 수, 조건부 git-only 문구,
  vehicle별 manifest 줄): 단순 값 토큰이 아니라 — ① 가변 블록은
  new-project.sh가 임시 파일에 멀티라인 블록으로 조립한 뒤, **자기 줄에 홀로
  있는 블록 토큰을 awk 마커-치환**으로 주입 ② manifest는 두 vehicle 줄을 모두
  가진 superset 템플릿에서 **해당 없는 줄을 삭제** (sed /패턴/d).

## 인터뷰 (커맨드 프로즈 — 전 항목 "왜 묻는지" 한 줄 + 추천 1개, 엔터=기본값)

| # | 질문 | 플래그 | 기본값 (추천) |
|---|---|---|---|
| 1 | 프로젝트 이름 (경로도) | `--name` (**필수**), `--dir` | dir 기본 `~/projects/<name>` — 프로즈가 사용자 관례(예: projects/ 배치)를 감지하면 그걸 제안; 스크립트 자체는 결정론적 기본만 |
| 2 | GitHub repo 만들까? | `--github <owner/name\|none>` | 만들기, **private** (공개는 명시 요청 시) |
| 3 | 트렁크 브랜치명 | `--trunk` | `main` |
| 4 | 캠페인 명명 | `--naming free\|numbered` | free |
| 5 | 병합 모델 | `--merge-model coordinator\|review-gate` | 1인=coordinator, 협업=review-gate |
| 6 | 트렁크 보호 훅? | `--hook\|--no-hook` | 협업/공유 repo면 hook, 개인 실험 repo면 no-hook |
| 7 | 외부 코드 반입 | `--host-name`, `--host-vehicle fork\|patches`, `--host-repo`, `--host-trunk`(fork만 필수) | 없음. 수정할 외부코드=fork, 패치 몇 장=patches |
| 8 | 노드 토폴로지: 단일 머신? 원격 계산 노드? 각 이름/역할/접속, **fs 공유 여부** | `--node <name:role[:ssh-alias]>` 반복, `--fs shared\|separate` | 단일 머신. 다중이면 fs=separate 가정 |
| 9 | 실행 환경(들): 하나? 프로그램/스테이지별로 갈리나? | `--env <name:uv\|conda\|module>[@machine]` 반복 — **첫 번째가 primary** | `main:uv` 하나 |
| 10 | 대용량 공유 데이터 | `--data-dir <path\|none>` | none |
| 11 | 배포 형태 | `--deploy none\|snapshot\|mirror` | none — 기록만 (도구는 ohd-share 백로그) |

고정 기본 (질문 아님): `WT_ROOT=$HOME/wt`, `STATE_DIR=docs/campaigns` —
드롭인 가이드와 동일; 바꾸려면 스캐폴드 후 campaign.sh 헤더 수동 편집.

인터뷰 원칙: 모호한 답은 재질문. 최종 실행 전 **요약 게이트**: 모인 플래그
전체 제시 → go 승인 → 스크립트 실행.

## `assets/new-project.sh` — 스캐폴드 순서와 산출물

**커밋 전략 (순서 고정)**: ① 전 산출물 생성 → ② **단일 스캐폴드 커밋**
(`chore: scaffold by oppenheimerdinger new-project`) → ③ `--hook`이면 그제서야
install-hooks.sh 실행 (훅은 스캐폴드 커밋 뒤 활성화 — docs-only 훅이 스캐폴드
자신을 막는 것 방지) → ④ `--github`면 repo 생성+push. 이후 사용자의 비-docs
커밋은 훅이 의도대로 막는다 (첫 캠페인 안내가 이를 설명).

**스크립트 결정론 기본값** (인터뷰 프로즈의 조건부 추천과 별개):
`--github none`, `--no-hook` — 프로즈가 인터뷰 답에 따라 명시 플래그를 넘긴다.
**Preflight (파일시스템 효과 전 fail-fast)**: `--name` 문법, `<dir>` 미존재
(존재하면 에러 — 부분 스캐폴드/재실행 미정의 문제 방지), `--data-dir` 실존,
env 조합 검증(uv는 primary 슬롯에서만 허용 — non-primary uv도 거부).

산출물 (`--name`만 필수, 나머지 기본값 내장):

1. `git init -b <trunk>` @ `<dir>`; `.gitignore`: `.omc/`, `.superpowers/`,
   `.venv/`, `__pycache__/`, `*.pyc`, `data` (심링크 — 머신 종속 절대경로라
   커밋 금지).
2. `tools/campaign.sh` — 헤더 기본값 치환 (TRUNK/NAMING/MERGE_MODEL +
   fork host면 DEP_DIR/DEP_TRUNK/PIN_FILE) + `docs/campaigns/.gitkeep`.
   **DEP_DIR 값 := `<dir 부모>/<host-name>`** (setup.sh의 기본 대상과 동일
   규칙; 런타임은 CAMPAIGN_DEP_DIR로 오버라이드).
3. `tools/install-hooks.sh` 복사 (실행은 커밋 전략 ③).
4. `CLAUDE.md` — 템플릿 인스턴스화.
5. `README.md` 스텁: 제목=이름, "goal: (한 줄)" 자리, "worked with the
   oppenheimerdinger harness" 한 줄.
6. `.claude/skills/.gitkeep`.
7. env(들): primary uv → `pyproject.toml`(`[project]` name/version/
   requires-python≥3.12만) + `.python-version`(3.12); primary conda →
   `environment.yml`(name/channels[conda-forge]/dependencies[python=3.12]);
   추가 conda → `envs/<name>.yml` 동일 스켈레톤; module → 파일 없음(매트릭스
   기록만). **uv는 primary 슬롯에서만 허용** (non-primary uv 포함 거부 — preflight).
8. host 지정 시 `hosts/<name>/`: manifest (`HOST/VEHICLE/REPO/PIN=`(빈 값,
   **양 vehicle 공통** — patches에선 패치가 적용되는 base SHA) + fork면
   `TRUNK` / patches면 `PATCHES=`(빈 값)) + setup.sh + README.md +
   `patches/.gitkeep`(빈 dir 소실 방지). 패치 파일 위치 `hosts/<name>/patches/`.
   **DEP_*·pin 배선은 fork 전용** — patches는 campaign.sh 헤더에 DEP_*를
   채우지 않음. 배선 값: `DEP_DIR := <dir 부모>/<host-name>`,
   `DEP_TRUNK := --host-trunk`, `PIN_FILE := hosts/<host-name>/manifest`.
9. `--data-dir` 지정 시: 경로 실존 검증(없으면 에러) → `data` 심링크 생성
   (.gitignore 처리, CLAUDE.md에 원 경로 기록; "read-only"는 규율 —
   원본 권한이 실제 보호선임을 매트릭스에 명기).
10. `--github owner/name`: `gh repo create --private --source=. --push`; gh
    부재/실패 시 정확한 수동 커맨드 출력 후 나머지는 완료 처리.

종료 출력: 구조 요약 + 다음 스텝: ① 새 세션을 repo에서 열기 ②
way-of-working 스킬 참조 ③ 첫 캠페인 `tools/campaign.sh new <이름>` —
**`--github none`이었다면 "campaign.sh는 origin 리모트가 필요 — 먼저
`git remote add origin <url> && git push -u origin <trunk>`" 안내를 ③ 앞에.**

## `assets/hosts-templates/` (umbrella-proj 패턴 일반화)

- **manifest**: `HOST= / VEHICLE=fork|patches / REPO= / TRUNK=(fork만) /
  PIN=(공통) / PATCHES=(patches만)`. KEY=VALUE — setup.sh와 campaign.sh pin이
  공동 소비.
- **setup.sh**: manifest 소스 → 대상 dir (`<HOST>_DIR` env > `<repo 부모>/
  <HOST>` 기본) → **FRESH**: clone 후 — PIN 있으면 checkout PIN; **PIN 비면
  fork는 TRUNK, patches는 기본 브랜치를 체크아웃하고 그 SHA를 출력하며
  "manifest의 PIN=에 기입하라" 안내** (첫 실행이 raw git 에러로 죽지 않음) —
  patches는 이어서 `hosts/<name>/patches/*.patch`를 `git apply` (의도적 dirty
  유지; **패치 0장이면 apply 단계를 스킵하고 안내 출력** — 갓 스캐폴드된 빈
  patches/에서 glob이 리터럴로 넘어가 죽는 것 방지) / **EXISTING**: adopt
  as-is, **HEAD 절대 이동 금지** / `--build`는 업스트림 빌드 스크립트에
  위임(있으면).
- **README.md**: "what we add" 자리.

## `assets/CLAUDE.md.template` (프로젝트 사실만)

`{{UPPER_SNAKE}}` 토큰. 섹션: ① `{{NAME}}` 한 줄 + goal 자리 ② 사실 블록 —
trunk/명명/병합모델/worktree root, **머신×환경 매트릭스** (행 = 머신: 이름/
역할/ssh/fs 공유; 그 아래 환경: 이름/타입/활성화 커맨드 자리. 다중 노드 +
fs=separate면 "코드 이동은 commit→push→pull, scp 금지" 인스턴스 문구 + 원격
노드 clone 안내 1줄), data 원경로, deploy 선택, hosts 목록 ③ 포인터: "작업
규율은 oppenheimerdinger 플러그인(way-of-working·campaign-land·
campaign-status); 라이프사이클은 `tools/campaign.sh`; 플러그인 미설치 세션은
플러그인 repo의 docs/campaign-dropin.md 참조" ④ 메모리 배선 1줄 ⑤ "이 파일은
claude-md-sanity 감사 대상 — 사실 변경 시 갱신" 자기유지 문구.

## 스모크 (tests/new-project-smoke.sh, CI 추가)

공통: 임시 dir, `--github none`, `CAMPAIGN_WT_ROOT`를 임시 dir로 export,
**`GIT_AUTHOR_NAME/EMAIL`·`GIT_COMMITTER_NAME/EMAIL` export** (new-project.sh가
스캐폴드 커밋을 새 repo 안에서 만들므로 CI의 무-identity 러너 대비).
프로필 1·2는 **throwaway bare origin을 만들어 remote add + trunk push 후**
캠페인 왕복 (campaign-smoke.sh:11–15 패턴).

1. **기본 프로필** (uv, hook, host 없음, **`--trunk work`** — 비-main 트렁크가
   훅/campaign.sh 헤더까지 치환되는지 겸사 검증): 구조 assert (git repo/트렁크명/
   CLAUDE.md 토큰 잔존 0/campaign.sh 헤더 값/pyproject/README/.gitkeep/
   .gitignore/install-hooks 존재) → `tools/campaign.sh new t1` → worktree+
   상태문서 → `abort t1` → **훅 검증: 신규 비-docs 파일**을 트렁크에서 커밋
   시도 → 거부 assert (스캐폴드 커밋 자체는 훅 활성화 전이라 통과했음도
   함께 assert).
2. **풀 프로필** (numbered, review-gate, no-hook, host **fork**, 다중 env
   (conda primary + 추가 conda + module@node), 다중 node + fs=separate,
   data-dir, **--deploy mirror**): hosts/ 파일 존재+토큰 치환,
   environment.yml + envs/<name>.yml, CLAUDE.md 매트릭스+git-only 문구+deploy
   기록, data 심링크 + .gitignore 반영, campaign.sh 헤더 DEP_*, numbered
   거부(`new badname`), non-primary uv 거부.
3. **경량 프로필** (host **patches**만): hosts/<name>/{manifest(PATCHES=,
   TRUNK 없음),setup.sh,patches/} 형태 + campaign.sh 헤더에 DEP_* **비어
   있음** assert. (스캐폴드만 — 캠페인 왕복 생략)
